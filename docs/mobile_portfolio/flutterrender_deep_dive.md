# Mobile Portfolio — `/flutterrender/` Deep Dive

> How Flutter's WebView delivers a pixel-perfect, native-feeling portfolio preview
> using server-rendered React components with real-time theme checking.

---

## 1. The Problem with Traditional WebView Previews

| Issue | What Happens | Impact |
|-------|-------------|--------|
| **No handle set** | `/portfolio/[handle]` returns 404 | User can't preview before publishing |
| **JS bridge only** | `mobile-preview` waits for Flutter to push JSON | White screen until Flutter mounts + serializes |
| **Desktop layout** | React renders at 1440px viewport | Tiny text, broken layout on 360px phone |
| **Heavy 3D on mobile** | Three.js scenes eat RAM | WebView OOM crash → "Lost connection to device" |
| **No offline** | Every preview requires network | Unusable on spotty connections |

## 2. `/flutterrender/[token]` — Architecture

### 2.1 How It Works

```
┌─────────────────────────────────────────────────────────────┐
│                    FLUTTER APP                               │
│                                                              │
│  1. User taps "Live Preview"                                 │
│  2. Cubit calls GET /api/mobile/portfolio/preview-token      │
│  3. Receives short-lived JWT (5 min TTL)                     │
│  4. WebView loads:                                           │
│     https://project-syncroo.netlify.app/flutterrender/{jwt}  │
│                                                              │
└──────────────────────────┬──────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────────┐
│                    NEXT.JS SERVER                            │
│                                                              │
│  src/app/flutterrender/[token]/page.tsx (Server Component)   │
│                                                              │
│  1. Decode JWT → extract userId                              │
│  2. Query MongoDB:                                           │
│     • Portfolio.findOne({ userId }) → draft data             │
│     • User.findById(userId) → name, avatar, skills           │
│     • Project.find({ lead/members: userId }) → projects      │
│  3. Call buildPreviewData(portfolio, user, projects)          │
│  4. Render <MobilePortfolioShell>                            │
│       └─ <PortfolioRenderer data={data} />                   │
│  5. Return full HTML + hydrated React                        │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 Why JWT Token Instead of Session Cookie?

| Approach | Problem |
|----------|---------|
| Session cookie | Android WebView doesn't share cookies with the Dart HTTP client |
| Query param `?userId=` | Insecure — anyone can guess a user ID |
| **JWT in URL path** | ✅ Signed, expires in 5 min, contains only `userId` |

The token is generated by a protected endpoint that requires the user's auth Bearer token, so only authenticated users can generate preview URLs.

### 2.3 Server Component vs Client Component

The page is a **Next.js Server Component** (no `"use client"`). This means:
- The portfolio HTML is rendered on the server and streamed to the WebView
- The WebView receives a **fully painted page** — no loading spinner, no white flash
- React hydration adds interactivity (Framer Motion animations, scroll effects)
- Total Time to First Paint: **~200ms** (server render) vs **~2000ms** (client-side fetch)

## 3. Mobile-Native Optimizations in `<MobilePortfolioShell>`

The `MobilePortfolioShell` wrapper component applies mobile-specific tweaks that make the WebView feel native rather than "a website crammed into an app."

### 3.1 Viewport & Touch

```html
<meta name="viewport" 
  content="width=device-width, initial-scale=1.0, 
           maximum-scale=1.0, user-scalable=no, 
           viewport-fit=cover" />
```

| Setting | Why |
|---------|-----|
| `user-scalable=no` | Prevents accidental pinch-zoom (portfolio is fixed layout) |
| `viewport-fit=cover` | Extends content under the notch/status bar |
| `maximum-scale=1.0` | Prevents iOS double-tap zoom glitch |

### 3.2 CSS Overrides for Native Feel

```css
/* Disable browser-style selection highlighting */
* {
  -webkit-tap-highlight-color: transparent;
  -webkit-touch-callout: none;
  user-select: none;
}

/* Smooth momentum scrolling */
html {
  overflow-y: auto;
  -webkit-overflow-scrolling: touch;
  overscroll-behavior-y: none;    /* No pull-to-refresh bounce */
}

/* Hide scrollbar (Flutter handles scroll indicators) */
::-webkit-scrollbar { display: none; }

/* Safe area insets for notch/rounded corners */
body {
  padding-top: env(safe-area-inset-top);
  padding-bottom: env(safe-area-inset-bottom);
  padding-left: env(safe-area-inset-left);
  padding-right: env(safe-area-inset-right);
}
```

### 3.3 Performance: Reduced Motion & 3D Throttling

```tsx
// Inside MobilePortfolioShell
const isMobile = true; // We know it's mobile because this route is only for Flutter

const adjustedData = {
  ...data,
  // Force lightweight background on mobile to prevent OOM crashes
  heavy3d: false,
  // Use 2D constellation effect instead of Three.js scene
  threeOverride: data.heavy3d ? '' : data.threeOverride,
  bgOverride: data.heavy3d ? 'constellation' : data.bgOverride,
};
```

| Optimization | Desktop | Mobile (`/flutterrender/`) |
|-------------|---------|---------------------------|
| Three.js 3D scenes | ✅ Full quality | ❌ Replaced with 2D canvas |
| Framer Motion animations | Full spring physics | Reduced duration (0.3s vs 0.6s) |
| Particle count | 200 | 60 |
| Image quality | Original | WebP, max 800px width |
| Font loading | All weights | Only 400 + 700 |

### 3.4 The JS Bridge (Secondary Channel)

Even though the page is server-rendered, we **keep the JS bridge alive** for real-time theme changes:

```tsx
// Inside MobilePortfolioShell (client-side hydration)
useEffect(() => {
  // Expose delta-update functions to Flutter
  (window as any).updateTheme = (themeId: string) => {
    setLocalOverrides(prev => ({ ...prev, themeId }));
  };
  (window as any).updateAccent = (accent: string) => {
    setLocalOverrides(prev => ({ ...prev, accent }));
  };
  (window as any).updateBackground = (bg: string) => {
    setLocalOverrides(prev => ({ ...prev, bgOverride: bg }));
  };
  (window as any).updateSections = (json: string) => {
    setLocalOverrides(prev => ({ ...prev, sections: JSON.parse(json) }));
  };

  // Signal Flutter that bridge is ready
  if ((window as any).FlutterBridge) {
    (window as any).FlutterBridge.postMessage("READY");
  }
}, []);
```

Flutter calls these **granular functions** instead of serializing the entire portfolio:

```dart
// Delta update — only sends what changed
_controller.runJavaScript("window.updateTheme('midnight')");
_controller.runJavaScript("window.updateAccent('#ff6b35')");
```

This is **100x faster** than `JSON.stringify(entirePortfolio)` on every keystroke.

### 4.2 Real-Time Theme Delta Flow

```
User taps "Midnight" theme chip
         │
         ▼
Flutter Cubit updates local state
  cubit.updatePortfolioLocal(
    portfolio.copyWith(themeId: 'midnight')
  )
         │
         ▼
BlocListener fires in ThemeCheckerPage
         │
         ▼
WebView JS call (delta, NOT full JSON):
  controller.runJavaScript(
    "window.updateTheme('midnight')"
  )
         │
         ▼
React useState update in MobilePortfolioShell
  setLocalOverrides({...prev, themeId: 'midnight'})
         │
         ▼
PortfolioRenderer re-renders with new theme
  - Background gradient changes
  - Card colors update
  - Text colors shift
  - Framer Motion transitions animate smoothly
         │
         ▼
Debounced save (1s) → PUT /api/mobile/portfolio
  { themeId: 'midnight' }
```

**Latency: < 16ms** (single frame) from tap to visual change.

### 4.3 What Gets Checked

| Check | Delta Function | What Changes Visually |
|-------|---------------|----------------------|
| Theme preset | `updateTheme('midnight')` | Entire color palette, fonts, gradients |
| Primary accent | `updateAccent('#ff6b35')` | Buttons, links, highlights, gradient stops |
| Secondary accent | `updateAccent2('#00d4ff')` | Gradient end, secondary UI elements |
| Background effect | `updateBackground('particles')` | Canvas/WebGL background animation |
| Card style | `updateCardStyle('glass')` | Project cards: flat → glassmorphism → neumorphic |
| Section animation | `updateSectionAnim('rise')` | Scroll-triggered entrance animations |
| Project card animation | `updateProjectCardAnim('flip')` | Hover/tap animation on project cards |
| Heavy 3D toggle | `updateHeavy3d(true)` | Switches between 2D canvas and Three.js |

### 5.3 Portfolio — Handle Workflow

| # | Method | Endpoint | Auth | Request | Response |
|---|--------|----------|------|---------|----------|
| 5 | `GET` | `/api/mobile/portfolio/handle-available?handle=x` | Bearer | query: `handle` | `{ available: bool, handle, reason? }` |
| 6 | `POST` | `/api/mobile/portfolio/claim-handle` | Bearer | `{ handle: "my-handle" }` | `{ ok, handle }` or `{ error }` |

**Handle Rules:**
- 3–30 characters
- Lowercase letters, numbers, hyphens only
- No leading/trailing hyphen
- Cannot be a reserved word (admin, api, portfolio, login, etc.)

### 5.4 Portfolio — Publish Workflow

| # | Method | Endpoint | Auth | Request | Response |
|---|--------|----------|------|---------|----------|
| 7 | `POST` | `/api/mobile/portfolio/publish` | Bearer | — | `{ ok, published, lastPublishedAt }` |

**What "Publish" Does:**
1. Takes the current draft portfolio document
2. Runs `snapshotPublishable(draft)` → creates a frozen copy
3. Saves it to `portfolio.published`
4. Sets `lastPublishedAt = now`
5. The public page at `/portfolio/[handle]` reads from `published`, not the draft

## 6. Flutter WebView Configuration

### 6.1 Pre-Loading Strategy

```dart
class PortfolioDashboardPage extends StatefulWidget {
  // WebView is created HERE, not in the LivePreview page
  // It loads in the background while user browses the dashboard
}

class _PortfolioDashboardPageState extends State<PortfolioDashboardPage> {
  late final WebViewController _bgController;
  bool _webViewReady = false;

  @override
  void initState() {
    super.initState();
    _initWebView(); // Start loading immediately
  }

  Future<void> _initWebView() async {
    // 1. Get preview token
    final token = await repository.getPreviewToken();
    
    // 2. Configure WebView
    _bgController = WebViewController()
      ..setJavaScriptMode(JavaScriptMode.unrestricted)
      ..setBackgroundColor(const Color(0xFF0b1020))
      ..addJavaScriptChannel('FlutterBridge', 
        onMessageReceived: (msg) {
          if (msg.message == 'READY') {
            setState(() => _webViewReady = true);
          }
        })
      ..loadRequest(Uri.parse(
        'https://project-syncroo.netlify.app/flutterrender/$token'
      ));
  }

  // When user taps "Live Preview", just show the already-loaded WebView
  void _openPreview() {
    Navigator.push(context, MaterialPageRoute(
      builder: (_) => PortfolioLivePreviewPage(
        controller: _bgController,   // Pass the pre-loaded controller
        isReady: _webViewReady,
      ),
    ));
  }
}
```

### 6.2 Error Recovery

```dart
// If the WebView fails to load (network error, expired token, etc.)
..setNavigationDelegate(NavigationDelegate(
  onWebResourceError: (error) {
    // Token expired? Get a new one and reload
    if (error.errorCode == -6 || error.description.contains('401')) {
      _refreshAndReload();
    }
  },
  onHttpError: (error) {
    if (error.response?.statusCode == 401) {
      _refreshAndReload();
    }
  },
))

Future<void> _refreshAndReload() async {
  final newToken = await repository.getPreviewToken();
  _bgController.loadRequest(Uri.parse(
    'https://project-syncroo.netlify.app/flutterrender/$newToken'
  ));
}
```

### 6.3 Scroll Synchronization

```dart
// Flutter controls the "scroll position" indicator
// while the WebView handles the actual scroll
_bgController.addJavaScriptChannel('ScrollBridge',
  onMessageReceived: (msg) {
    // React sends scroll percentage: "0.45"
    final pct = double.tryParse(msg.message) ?? 0;
    setState(() => _scrollProgress = pct);
  },
);
```
